2. Algoritma Pembentukan Garis

Garis adalah kumpulan titik. Algoritma pembentukan garis menentukan piksel-piksel mana yang harus diaktifkan untuk merepresentasikan garis lurus di layar raster. Persamaan garis lurus umum adalah y = mx + c, di mana m adalah gradien dan c adalah konstanta. Algoritma yang umum digunakan termasuk Brute Force, DDA (Digital Differential Analyzer), dan Bresenham.

Piksel yang digambar akan muncul di sini.

Prinsip Algoritma Garis:

Brute Force Algorithm:

Algoritma ini menggunakan persamaan garis y = mx + c secara langsung. Untuk setiap nilai x dari x_1 ke x_2, nilai y dihitung dan dibulatkan ke bilangan bulat terdekat untuk menentukan piksel. Kelemahan utamanya adalah penggunaan operasi floating-point dan pembulatan yang berulang, serta inefisiensi untuk garis vertikal/horizontal dan kemiringan tinggi. 

function drawLineBruteForce(x1, y1, x2, y2):
  dx = x2 - x1
  dy = y2 - y1

  if dx == 0: // Garis Vertikal
    for y from min(y1, y2) to max(y1, y2):
      drawPixel(x1, y)
    return

  m = dy / dx
  c = y1 - m * x1

  for x from min(x1, x2) to max(x1, x2):
    y = m * x + c
    drawPixel(x, round(y))

DDA (Digital Differential Analyzer) Algorithm:

Algoritma DDA memiliki kelebihan kecepatan dibanding Brute Force, terutama untuk gradien m > 1. Namun, DDA masih memerlukan pembulatan dan variabelnya tetap menggunakan angka real/floating point. 

function drawLineDDA(x1, y1, x2, y2):
  dx = x2 - x1
  dy = y2 - y1

  steps = max(abs(dx), abs(dy)) // Langkah total

  x_inc = dx / steps // Inkrementasi x per langkah
  y_inc = dy / steps // Inkrementasi y per langkah

  x = x1
  y = y1

  for i from 0 to steps:
    drawPixel(round(x), round(y))
    x = x + x_inc
    y = y + y_inc
Bresenham's Line Algorithm (0 < m < 1):

Dicetuskan oleh J.E. Bresenham pada tahun 1965, algoritma ini mengoptimalkan kualitas gambar garis di grafika raster karena tidak lagi memakai aritmetika floating-point, sehingga tidak diperlukan pembulatan. Iterasi dilakukan secara incremental. Kelemahannya, algoritma dasar ini hanya untuk kemiringan 0 < m < 1. 

function drawLineBresenham(x1, y1, x2, y2):
  dx = abs(x2 - x1)
  dy = abs(y2 - y1)
  p = 2 * dy - dx // Parameter keputusan awal

  x = x1
  y = y1

  for k from 0 to dx:
    drawPixel(x, y)
    if p < 0:
      p = p + 2 * dy
    else:
      p = p + 2 * dy - 2 * dx
      y = y + 1 // Untuk m>0
    x = x + 1

3. Poligon dan Filling

Poligon adalah gabungan dari beberapa garis lurus yang saling terhubung membentuk suatu luasan. Garis-garis ini disebut edge, dan titik pertemuan antar edge disebut vertex (verteks). Poligon dapat dibagi menjadi poligon sederhana (sisi tidak berpotongan, tidak ada lubang) dan tidak sederhana, serta poligon cembung (convex) dan non-cembung (non-convex).

Penambahan warna di dalam poligon disebut teknik area filling atau polygon filling. Dua cara umum adalah Boundary Fill Algorithm dan Scan Line Polygon Fill Algorithm.

Simulasi Scan Line Fill (Konseptual)

Teknik *Scan Line* mengisi poligon secara horizontal dari kiri ke kanan. Ini melibatkan pencarian titik perpotongan garis scan dengan edge poligon, mengurutkannya berdasarkan koordinat X, lalu mewarnai piksel di antara pasangan titik perpotongan tersebut. Aturan paritas ganjil-genap (odd-even rule) digunakan untuk menentukan apakah piksel berada di dalam atau di luar poligon.

Simulasi ini akan menyederhanakan proses filling dan tidak menampilkan setiap piksel yang diwarnai secara detail, tetapi akan menunjukkan area yang "diisi" secara konseptual. Implementasi penuh algoritma filling cukup kompleks.
